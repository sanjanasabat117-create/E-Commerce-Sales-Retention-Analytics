# Power BI Dashboard Setup Guide 📊

This guide provides step-by-step instructions on how to replicate the E-Commerce Analytics Dashboard using the synthetic data generated by this project. This acts as a comprehensive documentation of the **Data Modeling**, **DAX (Data Analysis Expressions)**, and **Visualization Design** skills demonstrated.

---

## 🏗️ 1. Data Import & Modeling (Star Schema)
1. Open Power BI Desktop and select **Get Data** > **Text/CSV**.
2. Import all six CSVs from the `data/` folder (`users.csv`, `products.csv`, `orders.csv`, `order_items.csv`, `sessions.csv`, `returns.csv`).
3. Navigate to the **Model View** and organize the tables into a **Star Schema**:
   * **Fact Tables (Center):** `Orders`, `Order Items`, `Returns`
   * **Dimension Tables (Surrounding):** `Users`, `Products`, `Sessions`
4. **Establish Relationships:**
   * `Users` (1) to `Orders` (*) via `user_id`
   * `Products` (1) to `Order Items` (*) via `product_id`
   * `Orders` (1) to `Order Items` (*) via `order_id`
   * `Orders` (1) to `Returns` (1/*) via `order_id`
   * `Sessions` (1) to `Orders` (1/*) via `session_id`

5. **Create a Professional Calendar Table (Recruiter Tip):** 
   * Go to the *Modeling* tab > *New Table* and enter:
     `Calendar = CALENDARAUTO()`
   * Connect `Calendar[Date]` (1) to `Orders[order_date]` (*) in the Model View.
   * **Important**: Always use the `Calendar` table for your Date Slicers and X-axis trends!

*(A star schema with a dedicated Date table optimizes performance and ensures accurate time-intelligence calculations like YoY Growth).*

## 🧮 2. Key DAX Measures (Best Practice)
Professional Power BI developers use a **single dedicated measures table** to keep business logic organized. 

1. **Create the Table**: Go to *Home > Enter Data*, name the table `_Measures`, and click Load.
2. **Move Measures**: Create the DAX formulas below inside this table. (Delete the default 'Column1' once you've added your first measure to turn it into a true "Measure Table" icon).
3. **Pro Tip (Display Folders)**: For extra recruiter "wow" factor, select your measures in the **Model View** and set a **Display Folder** (e.g., "Retention" or "Sales") in the Properties pane.

### Revenue & Profit
```dax
Total Revenue = SUMX('Order Items', 'Order Items'[quantity] * 'Order Items'[unit_price])

Total Cost = SUMX('Order Items', 'Order Items'[quantity] * RELATED('Products'[cost]))

Total Profit = [Total Revenue] - [Total Cost]

Profit Margin % = DIVIDE([Total Profit], [Total Revenue], 0)
```

### Orders & Conversion
```dax
Total Orders = DISTINCTCOUNT('Orders'[order_id])

Total Sessions = DISTINCTCOUNT('Sessions'[session_id])

Bounce Rate % = 
VAR Bounces = CALCULATE(DISTINCTCOUNT('Sessions'[session_id]), 'Sessions'[is_bounce] = 1)
RETURN DIVIDE(Bounces, [Total Sessions], 0)

Conversion Rate % = DIVIDE([Total Orders], [Total Sessions], 0)
```

### Return Analytics
```dax
Total Items Sold = SUM('Order Items'[quantity])

Total Items Returned = COUNT('Returns'[return_id])

Return Rate % = DIVIDE([Total Items Returned], [Total Items Sold], 0)
```

## 📈 3. Dashboard Layout & Visuals

The dashboard is structured into three distinct analytical pages:

### Page 1: Executive Overview
* **Cards (KPIs):** `Total Revenue`, `Total Profit`, `Profit Margin %`, `Total Orders`.
* **Line Chart:** `Total Revenue` (Y-axis) by `Orders[order_date]` (X-axis, Year/Month Hierarchy) to visualize seasonal trends.
* **Donut Chart:** `Total Revenue` broken down by `Products[category]`.
* **Slicers:** `Country` (from `Users`), `Date Range`.

### Page 2: Funnel & Conversion Analytics
* **Funnel Chart:** `Total Sessions` -> `Total Orders`.
* **Clustered Bar Chart:** `Conversion Rate %` and `Bounce Rate %` by `Sessions[device]`.
* **Scatter Plot:** `Total Revenue` (Y-axis) vs. `Sessions[duration_seconds]` (X-axis) to see if longer sessions result in higher value carts.

### Page 3: Operations & Returns
* **Matrix Visual:** `Products[category]` (Rows), `Products[product_name]` (Sub-rows), showing `Total Items Sold`, `Total Items Returned`, and `Return Rate %`.
* **Tree Map:** `Returns[reason]` (Groups) and `Total Items Returned` (Values) to instantly identify the most common qualitative issues (e.g., "Defective", "Wrong Size").

### 🎨 Design Best Practices Implemented:
* **Theming:** A clean, minimalist corporate theme using a distinct color palette (e.g., cool blues for revenue, standard grays for backgrounds).
* **Interactivity:** Ensured cross-filtering is enabled (e.g., clicking on "Electronics" filters the funnels and KPIs).
* **Usability:** Used clear headers and categorized DAX measures for maintainability.
